Phase 1: 子领域扩展

从 6 大领域生成细分研究方向与搜索关键词

▼
输入→6 个硬编码研究大领域→输出 ~30 个子领域（含搜索关键词）
核心逻辑
  • LLM 为每个大领域生成 N 个具体、当前活跃的研究子领域
  • 子领域数量 N 由目标 query 总数反推，保证漏斗产量足够
  • 每个子领域附带 3-5 个 Semantic Scholar 搜索关键词
  • 要求子领域足够具体（如「检索增强生成」而非「NLP」）
🔍

Phase 2: 论文发现与作者定位

通过 Semantic Scholar API 找到高引论文和活跃研究者

▼
输入→子领域 + 关键词→输出 每个子领域的论文池
两阶段时间窗口
作者发现窗口近 36 个月
论文采集窗口近 18 个月

先用 2-3 年窗口找高引论文（确保引用量充足），提取活跃作者后，再拉取他们近 12-18 个月的最新发表（确保前沿性）。

核心逻辑
  • 按引用速率（citations / 论文月龄）排序，避免偏向旧论文
  • 统计高频出现的作者 → 取 top 5 活跃研究者
  • 拉取每位作者的近期发表 → 汇总为子领域论文池
关键参数
top 论文 / 子领域20
top 作者 / 子领域5
论文 / 作者10
🔬

Phase 3: 知识提取 + Layer 1 验证

从论文摘要中提取结构化知识，并验证提取准确性

▼
输入→论文池（含摘要）→输出 通过验证的结构化知识库
提取内容（LLM Call #1）
  • research_question — 核心研究问题
  • methodology — 方法论概述
  • key_findings — 关键发现列表
  • limitations — 研究局限
  • claims — 具体可验证声明（含类型标签）
Layer 1 验证逻辑（LLM Call #2）
  • 将提取结果与原始摘要交给另一次 LLM 调用做对比审查
  • 通过条件：accuracy ≥ 0.8 且无 critical 级别错误
  • 未通过验证的论文直接移除，不进入后续阶段
🔗

Phase 4: 关系挖掘 + Layer 2 验证

发现论文间的方法论争论、矛盾、互补等非平凡关系

▼
输入→已验证知识库→输出 已验证的论文关系组
五种关系类型
  • 方法论争论 — 不同方法研究同一问题
  • 矛盾解决 — 论文结论相互冲突
  • 文献综合 — 覆盖同一主题的不同方面
  • 定量验证 — 可交叉引用的量化结果
  • 跨领域桥梁 — 不同子领域的方法可互相借鉴
Layer 2 验证逻辑
  • 将检测到的关系连同论文摘要交给 LLM 做二次确认
  • 核心问题：「这些论文是否真的在研究同一问题？矛盾是否真实存在？」
  • 通过条件：verdict = verified 且 confidence ≥ 0.7
✍

Phase 5: 查询生成 + Layer 3 验证

基于论文关系生成高难度研究问题，同时生成参考答案

▼
输入→已验证关系组→输出 Query + 参考答案 + 源论文
生成内容
  • query — 30-80 词研究问题（有明确交付物与时间范围）
  • reference_answer — 500-800 词参考答案（引用源论文）
  • expected_answer_elements — 完整回答应包含的要素清单
  • difficulty_factors — 难度来源分析
Layer 3 验证逻辑
  • 检查参考答案中引用的论文是否在源论文列表中
  • 检查归因到特定论文的声明是否与该论文摘要一致
  • 通过条件：verification_passed = true 且 score ≥ 0.6
🛠

Phase 6: 过滤与质量控制

搜索必要性验证 → 难度下限 → 去重 → 策略均衡

▼
四道过滤门
  • 搜索必要性测试 — LLM 在无搜索条件下回答 → 评估基线质量 → 淘汰基线已足够好的问题（quality_score ≤ 0.7 才保留）
  • 难度下限 — 需引用论文 ≥ 3 篇，难度因子 ≥ 2 项
  • 语义去重 — Jaccard 相似度 > 0.6 视为重复
  • 策略均衡裁剪 — 按权重分配配额，裁剪至目标数量
基线评估五维度
准确性accuracy
完整性completeness
深度depth
时效性timeliness
具体性specificity

核心逻辑：如果 LLM 不搜索就能给出高质量回答 → 该问题不需要深度搜索 → 淘汰。

数据锚点：真实市场事件 + 多源金融数据

初步思路

目前的想法是从 SEC EDGAR 中获取近期上市公司的重大事件披露（8-K 文件，如财报意外、并购公告、管理层变更），结合 FRED 的宏观经济指标（利率、CPI、就业数据等）构建"宏观 → 行业 → 个股"的多层推理场景。通过财经新闻 API 识别近期市场争议性事件（如估值分歧、监管冲击），再拉取相关公司的实际财务数据（营收、利润率、估值倍数）作为 query 中可验证的定量锚点，确保生成的问题需要多源数据交叉分析才能回答。

医生 / 医疗人员 初步方案

数据锚点：临床试验 + 冲突性指南 + 患者约束

▼
初步思路

目前的想法是从 PubMed 中搜索近 1-2 年发表的高影响力随机对照试验（RCT）和 Meta 分析，特别关注那些可能改变临床实践的大型试验；同时从 FDA/EMA 的审批数据库获取近期新获批药物（证据仍在积累的窗口期）。关键创新点是挖掘不同医学学会（如 AHA vs ESC、ADA vs EASD）对同一疾病的指南推荐分歧，再叠加真实的患者约束条件（合并症、禁忌症、经济因素），生成需要在循证证据与临床现实之间做权衡的 query。

数据源
PubMed E-utilities API 免费 ClinicalTrials.gov API 免费 OpenFDA API 免费 WHO / CDC 指南 免费 Cochrane Library 部分免费
4 种 Query 策略
⚖ 指南冲突裁判
分析不同权威指南（ADA vs EASD、AHA vs ESC）对同一问题的分歧原因
对于 HbA1c 在 7-8% 之间的 2 型糖尿病患者，ADA 2025 指南推荐的一线联合用药方案与 EASD 的推荐有何不同？对于同时有 CKD Stage 3 的患者，哪个方案更有证据支持？
💊 新药循证评估
评估新获批药物的证据质量、临床意义和适用人群
Lecanemab（Leqembi）获 FDA 完全批准用于早期阿尔茨海默病。基于 Clarity AD 试验数据，其临床获益是否达到临床意义阈值？ARIA 发生率如何？哪些患者亚群获益最大？
🩺 复杂病例决策
在多种合并症/禁忌症约束下制定个体化治疗方案
一位 68 岁 2 型糖尿病患者，合并 CKD G3b 期（eGFR 35）和心衰（EF 40%）。根据最新 ADA/KDIGO 指南，如何调整降糖方案？需权衡肾功能保护、心血管获益和低血糖风险
📈 试验数据交叉验证
核验具体临床试验的数字、安全性数据和亚组分析结果
DAPA-HF、EMPEROR-Reduced 和 DELIVER 三项大型 RCT 中，SGLT2 抑制剂对心衰患者主要终点的风险比分别是多少？不同射血分数分层下结果是否一致？

律师 / 法研人员 初步方案

数据锚点：真实法律争议 + 法源引用链 + 跨法域规则

▼
初步思路

目前的想法是从 CourtListener 获取近期联邦法院的里程碑判决，通过引用链追踪找到对同一法律原则存在不同裁判立场的案件（如巡回法院间的 circuit split）；同时从 congress.gov 和 EUR-Lex 获取近期通过的新立法（如 AI Act、数据隐私法规），识别法律条文中存在解释空间的模糊条款。关键创新点是自动挖掘同一法律问题在不同法域（美国/欧盟/中国）的规则差异，构建需要精确引用法源、做跨法域对比分析的 query。
